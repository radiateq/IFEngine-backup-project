                        Louis Langholtz iforce2d • a year ago

                        Box2D - at least as of 2.3.2 - unfortunately doesn't give the user access to 100% of all the data necessary to entirely serialize and deserialize a world in any manner however. So while re-creating a world will get most state, it won't be a perfect recreation. Take the b2Body::m_sleepTime member variable for example. Without this info, a deserialized restart/clone/recreation won't come to rest as soon as it otherwise might.

                                Louis Langholtz Antonio Manuel • a year ago

                                Antonio Manuel Your request for cloning the world is one I like a lot. I'd also like to see it in Box2D. In the meantime though, I've added this capability now to PlayRho (my fork of Box2D). See https://github.com/louis-la... . Cloning is done idiomatically via C++ copy construction of a world instance or via C++ copy assignment. I.e. one can simply write code like `World worldCopy = world;`. And this copies all state data - even the sleep time, broad phase, and contact warm starting data. So these should be 100% exact copies.
